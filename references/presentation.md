# Presentation Workflow

Use this when turning the LLM Wiki process into a deck, video outline, or prompt for Kimi/Claude/Gamma.

## Narrative

Recommended story:

1. Hook: The user thought they were making a beautiful wiki for humans.
2. Realization: LLM Wiki is mainly for AI agents to read.
3. First import: notes become concepts and the agent answers with better context.
4. Second import: AI memory exports merge scattered self-knowledge.
5. Expansion: media, music, browsing, health, tools, and lifestyle data make the agent more grounded.
6. Result: the wiki becomes a context home for the agent.
7. Boundary: private full wiki, public redacted story.

## Slide Pattern

Use 12-14 slides for a 3-5 minute video:

- 1 title slide
- 2-3 setup slides
- 6-8 source/capability slides
- 1 privacy slide
- 1 closing slide

Each slide should answer:

- What data was imported?
- What did it teach the agent?
- What can the agent do now?

## Visuals

Prefer:

- warm off-white background
- large human-readable titles
- three card layout
- simple source icons
- clean graph/map diagrams
- aggregate numbers only

Avoid:

- raw screenshots of private data
- messy graph view screenshots
- huge tables
- dark cyberpunk styling unless requested
- generic "AI changes everything" copy

## Privacy-Safe Public Version

Use:

- categories, counts, date ranges, high-level patterns
- synthetic examples
- screenshots of mock pages

Avoid:

- raw URLs
- personal family details
- exact health readings
- account IDs
- unredacted email or platform exports
